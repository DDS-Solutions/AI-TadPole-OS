## 🎯 Primary Objective
Perform deep architectural scouting of external repositories to extract patterns, vulnerabilities, or innovation vectors for the Tadpole Registry.

## 🏗️ Scouting Protocol

### 1. Discovery Phase
- **Target Identification**: Search for the exact repository handle or topic.
- **Vitals Check**: Analyze star count, fork velocity, and recent commit frequency.

### 2. Structural Decomposition
- **File System Audit**: Map the `src/` hierarchy and identify the core entry point.
- **Logic Mapping**: Read `ARCHITECTURE.md` or `README.md` to identify the data flow.

### 3. Aletheia Synthesis (Intelligence Dossier)
- **Innovation Vector**: What does this repo do better than the current registry?
- **Risk Analysis**: Are there licensing (GPL) or security conflicts?
- **Hardening Map**: How would this repo fit into the **4-Pillars** framework?

### 4. Intelligence Sharing
- Call `share_finding` with the topic "Registry Intelligence".
- Generate a Research Dossier in the `.tmp/scout/` directory.
