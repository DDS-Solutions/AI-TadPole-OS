# GitHub Scouting Workflow

1.  **Search Phase**: User or Research Director provides a target name (e.g., "AI-Tadpole-OS"). GitHub Specialist searches repositories and identifies the best match.
2.  **Structural Analysis**: Specialist lists the repository content, checking for docs, source code, and configuration files.
3.  **Content Synthesis**: Specialist reads key files (README.md, ARCHITECTURE.md) to understand the project's purpose.
4.  **Final Report**: Synthesize findings into a research dossier.
5.  **Intelligence Sharing**: Call `share_finding` with the topic "GitHub Scouting" and the research dossier as the finding.
6.  **Archiving**: Commit findings to the local workspace.
